1. Log Levels

LevelValueUse Case
DEBUG 10Detailed diagnostic info during development
INFO 20Confirmation that things are working as expected
WARNING 30Something unexpected happened, but the app continues
ERROR 40A serious problem; a function failed
CRITICAL 50A severe error; the program may not continue
Only messages at or above the configured level are emitted. Setting a logger to WARNING silences DEBUG and INFO.

2. Basic Setup

import logging

logging.basicConfig(
    level=logging.DEBUG,
    format="%(asctime)s [%(levelname)s] %(name)s - %(message)s",
    datefmt="%Y-%m-%d %H:%M:%S"
)

logger = logging.getLogger(__name__)

logger.debug("Debug message")
logger.info("Info message")
logger.warning("Warning message")
logger.error("Error message")
logger.critical("Critical message")

Output:

2026-05-02 10:00:00 [DEBUG]    __main__ - Debug message
2026-05-02 10:00:00 [INFO]     __main__ - Info message
2026-05-02 10:00:00 [WARNING]  __main__ - Warning message
2026-05-02 10:00:00 [ERROR]    __main__ - Error message
2026-05-02 10:00:00 [CRITICAL] __main__ - Critical message
Rule: Call basicConfig() only once, at the application entry point (main.py / app.py). Calling it multiple times has no effect after the first call.

3. Logger Hierarchy

Loggers form a dotted-name tree. A logger named myapp.db is a child of myapp, which is a child of the root logger. Log records propagate up the tree unless propagate = False. 

# Root logger (avoid using directly)
root = logging.getLogger()

# Best practice: always use __name__
logger = logging.getLogger(__name__)   # e.g. "myapp.services.user"

# Child logger with its own level
db_logger = logging.getLogger("myapp.db")
db_logger.setLevel(logging.WARNING)       # quieter than parent

# Prevent propagation to root
audit_logger = logging.getLogger("audit")
audit_logger.propagate = False

import logging
from logging.handlers import RotatingFileHandler, TimedRotatingFileHandler

logger = logging.getLogger("myapp")
logger.setLevel(logging.DEBUG)

fmt = logging.Formatter("%(asctime)s [%(levelname)s] %(name)s - %(message)s")

# Console — INFO and above
console = logging.StreamHandler()
console.setLevel(logging.INFO)
console.setFormatter(fmt)

# Rotating file — DEBUG and above, max 5 MB, keep 3 backups
file_h = RotatingFileHandler("app.log", maxBytes=5_000_000, backupCount=3)
file_h.setLevel(logging.DEBUG)
file_h.setFormatter(fmt)

# Daily rotation, keep 7 days
timed_h = TimedRotatingFileHandler("daily.log", when="midnight", backupCount=7)
timed_h.setFormatter(fmt)

logger.addHandler(console)
logger.addHandler(file_h)
logger.addHandler(timed_h)

5. Structured / JSON Logging Production Standard

Plain-text logs are hard to query. JSON logs are natively parsed by Elasticsearch, Splunk, CloudWatch, Datadog, and most modern log platforms. 

import logging, json

class JSONFormatter(logging.Formatter):
    def format(self, record):
        log = {
            "timestamp": self.formatTime(record),
            "level":     record.levelname,
            "logger":    record.name,
            "message":   record.getMessage(),
            "module":    record.module,
            "line":      record.lineno,
        }
        if record.exc_info:
            log["exception"] = self.formatException(record.exc_info)
        # Merge any extra= fields passed by the caller
        _std = logging.LogRecord.__dict__
        for k, v in record.__dict__.items():
            if k not in _std and not k.startswith("_"):
                log[k] = v
        return json.dumps(log)

handler = logging.StreamHandler()
handler.setFormatter(JSONFormatter())

logger = logging.getLogger("myapp")
logger.addHandler(handler)
logger.setLevel(logging.DEBUG)

# Extra context attached inline
logger.info("User logged in", extra={"user_id": 42, "ip": "10.0.0.1"})

Output:

{"timestamp": "2026-05-02 10:00:00,000", "level": "INFO", "logger": "myapp",
 "message": "User logged in", "module": "app", "line": 28,
 "user_id": 42, "ip": "10.0.0.1"}

6. Configuration via File

Hard-coding logger setup in Python clutters app code and makes it hard to adjust levels per environment. Use dictConfig loaded from YAML or JSON instead.

logging.yaml

version: 1
disable_existing_loggers: false

formatters:
  standard:
    format: "%(asctime)s [%(levelname)s] %(name)s - %(message)s"
  json:
    "()": myapp.logging_utils.JSONFormatter

handlers:
  console:
    class: logging.StreamHandler
    level: INFO
    formatter: standard
    stream: ext://sys.stdout
  file:
    class: logging.handlers.RotatingFileHandler
    level: DEBUG
    formatter: json
    filename: logs/app.log
    maxBytes: 10485760   # 10 MB
    backupCount: 5

root:
  level: DEBUG
  handlers: [console, file]

loggers:
  myapp.db:          # silence noisy DB logger in production
    level: WARNING
    propagate: true

Load in Python

import logging.config
import yaml

with open("logging.yaml") as f:
    config = yaml.safe_load(f)

logging.config.dictConfig(config)
logger = logging.getLogger(__name__)

Database Query Logging

import time

db_logger = logging.getLogger("myapp.db")

def execute_query(sql, params):
    db_logger.debug("Executing query", extra={"sql": sql, "params": params})
    start = time.perf_counter()
    try:
        result = conn.execute(sql, params)
        elapsed = time.perf_counter() - start

    except Exception:
        db_logger.exception("Query failed", extra={"sql": sql})
        raise

PitfallFix
Using print() instead of loggingAlways use logger.* in production code
Using the root logger directlyAlways getLogger(__name__)
Calling basicConfig() multiple timesCall once at the app entry point only
Logging inside tight loopsGuard with logger.isEnabledFor(logging.DEBUG)
Missing tracebacks on exceptionsUse logger.exception() or exc_info=True
Logging PII / passwordsScrub or mask sensitive fields before logging

Performance Guard for Expensive Log Messages

# BAD — f-string is always evaluated, even when DEBUG is disabled
logger.debug(f"State: {expensive_serialize(obj)}")

# GOOD — evaluated only when the level is actually active
if logger.isEnabledFor(logging.DEBUG):
    logger.debug(f"State: {expensive_serialize(obj)}")

10. Debugging Production Issues Using Logs Incident Response

Production bugs rarely come with a debugger attached. Logs are your only visibility. Below are concrete patterns used in real incident investigations.

Strategy 1 — Correlate by Request ID

Without a correlation ID you cannot follow a single request through multiple services. Set a unique ID at the entry point and stamp every log line with it. 

# middleware.py — FastAPI
import uuid, logging
from contextvars import ContextVar
from starlette.middleware.base import BaseHTTPMiddleware

request_id_var: ContextVar[str] = ContextVar("request_id", default="-")

class RequestIDMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request, call_next):
        # Accept from upstream (API gateway / load balancer) or generate
        rid = request.headers.get("X-Request-ID", str(uuid.uuid4()))
        request_id_var.set(rid)
        response = await call_next(request)
        response.headers["X-Request-ID"] = rid   # echo back to caller
        return response

class RequestIDFilter(logging.Filter):
    def filter(self, record):
        record.request_id = request_id_var.get()
        return True

# Attach filter to your logger so every line has request_id
logging.getLogger("myapp").addFilter(RequestIDFilter())
# Format: "%(asctime)s [%(request_id)s] [%(levelname)s] %(message)s"

# Kibana / Splunk query to replay one request end-to-end:
# request_id:"abc-123-xyz"  →  all log lines in exact order

Strategy 2 — Breadcrumb Decorator

Log entry, exit, duration, and any exception for key functions without touching their internals. Pinpoints exactly which layer is slow or failing. 

import time, logging, functools

def log_call(logger):
    """Decorator: logs entry, exit, duration, and exceptions."""
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            logger.debug(f"{func.__name__} ▶ called")
            t0 = time.perf_counter()
            try:
                result = func(*args, **kwargs)
                ms = round((time.perf_counter() - t0) * 1000, 2)
                logger.debug(f"{func.__name__} ✔ OK", extra={"elapsed_ms": ms})
                return result
            except Exception as exc:
                ms = round((time.perf_counter() - t0) * 1000, 2)
                logger.exception(f"{func.__name__} ✖ FAILED",
                               extra={"elapsed_ms": ms, "error": str(exc)})
                raise
        return wrapper
    return decorator

log = logging.getLogger("myapp.payment")

@log_call(log)
def charge_card(user_id, amount):
    ...

@log_call(log)
def send_receipt(user_id, order_id):
    ...

# Log output during an incident:
# [abc-123] charge_card ▶ called
# [abc-123] charge_card ✖ FAILED  elapsed_ms=1203  error="Gateway timeout"
# → immediately isolates: charge_card failed, send_receipt never ran

Strategy 3 — Log State Before a Known Failure Point

When a bug is reported ("fails only for user 99"), log full context around the suspect path so the next occurrence captures everything you need. 

def apply_discount(user_id, cart):
    logger.debug("apply_discount: entering", extra={
        "user_id":    user_id,
        "cart_items": len(cart.items),
        "cart_total": cart.total,
        "has_coupon": cart.coupon is not None,
        "coupon":     cart.coupon,
    })
    try:
        discount = calculate_discount(cart)
        logger.info("Discount applied",
                   extra={"user_id": user_id, "discount": discount})
        return discount
    except ValueError as e:
        # Full context — no guessing in post-mortem
        logger.error("Discount calculation failed", extra={
            "user_id":    user_id,
            "coupon_code": cart.coupon,
            "cart_total": cart.total,
            "error":      str(e),
        })
        raise

Strategy 4 — Slow-Query / Latency Detection

Emit a WARNING whenever any operation exceeds a threshold. Alerting rules can then trigger on the log pattern — no APM agent required. 

SLOW_MS = 500
perf_log = logging.getLogger("myapp.perf")

def timed(name, fn, *args, **kwargs):
    t0 = time.perf_counter()
    result = fn(*args, **kwargs)
    ms = (time.perf_counter() - t0) * 1000
    level = logging.WARNING if ms > SLOW_MS else logging.DEBUG
    perf_log.log(level, "op finished", extra={
        "op":         name,
        "elapsed_ms": round(ms, 2),
        "slow":       ms > SLOW_MS,
    })
    return result

# Usage:
rows = timed("db.get_active_users", db.get_active_users, org_id=7)

# Splunk / CloudWatch Insights alert rule:
# slow:true AND elapsed_ms > 500  →  PagerDuty / Slack alert

Strategy 5 — Structured Error Fingerprinting

Include a stable error_code in every error log. Group, count, and alert on specific error types without fragile text-regex matching. 

class AppError(Exception):
    def __init__(self, message, code):
        super().__init__(message)
        self.code = code   # e.g. "PAYMENT_DECLINED", "USER_NOT_FOUND"

def process_payment(order):
    try:
        _charge(order)
    except AppError as e:
        logger.error("Payment failed", extra={
            "error_code": e.code,       # stable, queryable fingerprint
            "order_id":   order.id,
            "amount":     order.amount,
            "currency":   order.currency,
        })
        raise

# Incident queries:
# error_code:PAYMENT_DECLINED | timechart count  → spike chart
# error_code:PAYMENT_DECLINED | stats count by user_id  → blast radius
# error_code:PAYMENT_DECLINED AND amount > 1000  → high-value failures

End-to-End Incident Workflow

#StepWhat You DoExample Log Query
1Detect Alert fires on elevated ERROR rate level:ERROR | timechart count
2Scope Find the error type and affected users error_code:PAYMENT_DECLINED | stats count by user_id
3Trace Pick one affected request and replay it request_id:"abc-123-xyz"
4Isolate Find the exact function / layer that failed request_id:"abc-123-xyz" AND level:ERROR
5Root-cause Read stack trace + surrounding context fields request_id:"abc-123-xyz" | sort @timestamp
6Verify fix Confirm error rate drops after deploy error_code:PAYMENT_DECLINED | timechart count
Never log a raw exception without context. Always include identifiers (user ID, order ID, request ID). An exception message tells you what failed — the extra fields tell you who, when, and how often.
Security reminder: Never log passwords, API keys, full credit card numbers, or other PII. Mask or redact before writing to any log sink.

main.py — App Factory with Middleware

# app/main.py
import uuid, logging, time
from contextvars import ContextVar
from fastapi import FastAPI, Request
from fastapi.responses import JSONResponse
from app.logging_config import setup_logging

setup_logging()   # ← must be first, before any logger is created

request_id_var: ContextVar[str] = ContextVar("request_id", default="-")
logger = logging.getLogger("app")

app = FastAPI()

@app.middleware("http")
async def logging_middleware(request: Request, call_next):
    rid = request.headers.get("X-Request-ID", str(uuid.uuid4()))
    request_id_var.set(rid)
    t0 = time.perf_counter()

    logger.info("Request started", extra={
        "request_id": rid,
        "method":     request.method,
        "path":       request.url.path,
        "client_ip":  request.client.host,
    })

    response = await call_next(request)
    ms = round((time.perf_counter() - t0) * 1000, 2)

    level = logging.WARNING if response.status_code >= 500 else logging.INFO
    logger.log(level, "Request finished", extra={
        "request_id":  rid,
        "status_code": response.status_code,
        "elapsed_ms":  ms,
    })

    response.headers["X-Request-ID"] = rid
    return response

@app.exception_handler(Exception)
async def global_exception_handler(request: Request, exc: Exception):
    logger.exception("Unhandled exception", extra={
        "request_id": request_id_var.get(),
        "path":       request.url.path,
    })
    return JSONResponse(status_code=500,
                         content={"detail": "Internal server error"})

12. Centralized Logging: ELK Stack Elasticsearch Logstash Kibana

The ELK stack (Elasticsearch + Logstash + Kibana) — or its modern variant Elastic Stack with Filebeat — is the most widely used centralized log platform in production Python deployments. The Python app writes structured JSON logs; a shipper forwards them to Elasticsearch; Kibana provides search, dashboards, and alerting.

Architecture Overview

ComponentRoleWhat you configure
Python app Produces JSON log lines JSON formatter, RotatingFileHandler or stdout
Filebeat Lightweight log shipper (runs as sidecar / daemon) Which files/containers to tail; output → Logstash or ES directly
Logstash Parse, enrich, filter, route Input (beats), filter (json codec, grok), output (elasticsearch)
Elasticsearch Stores & indexes logs; full-text + field queries Index lifecycle policy (ILM), mappings for typed fields
Kibana Visualize, search, alert Index patterns, dashboards, Watcher/alert rules
Data flow:
Python app → writes JSON to stdout / file → Filebeat ships to Logstash → parses & enriches → Elasticsearch index → Kibana dashboards & alerts

Step 1 — Python: Emit Clean JSON to stdout

Containers (Docker/K8s) read stdout. Write one JSON object per line; Filebeat collects it without any file path config. 

# app/logging_utils.py
import logging, json, traceback

class ELKFormatter(logging.Formatter):
    """One JSON object per line — ECS (Elastic Common Schema) compatible."""
    def format(self, record):
        doc = {
            "@timestamp":    self.formatTime(record, "%Y-%m-%dT%H:%M:%S.%fZ"[:-1]),
            "log.level":     record.levelname.lower(),
            "log.logger":    record.name,
            "message":       record.getMessage(),
            "service.name":  "identity-service",
            "process.pid":   record.process,
            "log.origin": {
                "file.name": record.filename,
                "function":  record.funcName,
                "line":      record.lineno,
            },
        }
        if record.exc_info:
            doc["error.stack_trace"] = self.formatException(record.exc_info)
            doc["error.type"] = record.exc_info[0].__name__ if record.exc_info[0] else None
        # Merge caller-supplied extra= fields at top level
        _skip = logging.LogRecord.__dict__
        for k, v in record.__dict__.items():
            if k not in _skip and not k.startswith("_"):
                doc[k] = v
        return json.dumps(doc, default=str)
The field names above follow Elastic Common Schema (ECS) conventions (@timestamp, log.level, service.name, error.stack_trace). Using ECS means Kibana's built-in dashboards and ML jobs recognise your fields without custom mappings.

# logstash/pipeline/python-logs.conf
input {
  beats { port => 5044 }

filter {
  # Message is already JSON from ELKFormatter — just parse it
  if [message] =~ /^\{/ {
    json {
      source => "message"
      skip_on_invalid_json => true
    }
  }

  # Normalise timestamp field to @timestamp
  if [@timestamp] {
    date {
      match => ["@timestamp", "ISO8601"]
      target => "@timestamp"
    }
  }

  # Tag slow requests automatically
  if [elapsed_ms] and [elapsed_ms] > 500 {
    mutate { add_tag => ["slow_request"] }
  }

  # Drop health-check noise
  if [path] == "/health" or [path] == "/metrics" {
    drop { }
  }
}

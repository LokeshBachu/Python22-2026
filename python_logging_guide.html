Python Logging

End-to-end reference — stdlib logging module

1 · Introduction

Python's built-in logging package (PEP 282) provides a flexible, thread-safe logging framework that covers everything from a quick print replacement to enterprise-grade, structured, multi-destination logging pipelines — all without third-party dependencies.

  • Part of the Python standard library since Python 2.3
  • Thread-safe; partially multiprocess-safe with QueueHandler
  • Configurable at runtime without code changes (via dictConfig / fileConfig)
  • Hierarchical logger namespace that mirrors the package tree

2 · Architecture & Data Flow

Every log call travels through these five stages:

Loggers form a hierarchy separated by dots. A child logger propagates records up to its parent unless propagate = False. The root logger (logging.getLogger()) sits at the top.

# Hierarchy example
"myapp"        # parent
"myapp.db"     # child — propagates to "myapp" by default
"myapp.db.sql" # grandchild

3 · Log Levels

LevelNumeric ValueUse when…
DEBUG 10 Detailed diagnostic information (dev / troubleshooting only)
INFO 20 Confirmation that things are working as expected
WARNING 30 Something unexpected, but the app continues (default level)
ERROR 40 A serious problem; some function could not be performed
CRITICAL 50 A severe error; the program itself may be unable to continue 
import logging

logger = logging.getLogger(__name__)

logger.debug("x = %s", x)
logger.info("Server started on port %d", port)
logger.warning("Config missing; using defaults")
logger.error("Failed to connect to database")
logger.critical("Disk full — shutting down")

# Or with explicit level:
logger.log(logging.INFO, "Custom level call")
Use lazy % formatting (logger.debug("val=%s", val)) not f-strings — the string is only built if the message actually gets emitted.

4 · Quick Start with basicConfig

logging.basicConfig() is a one-call setup for the root logger. It has no effect if the root logger already has handlers attached.

import logging

# Console only (default)
logging.basicConfig(level=logging.DEBUG)

# File + format
logging.basicConfig(
    filename="app.log",
    filemode="a",                       # "w" to overwrite each run
    level=logging.INFO,
    format="%(asctime)s [%(levelname)s] %(name)s — %(message)s",
    datefmt="%Y-%m-%d %H:%M:%S",
    encoding="utf-8",                   # Python ≥ 3.9
)

# Both console AND file (Python ≥ 3.8 — handlers kwarg)
logging.basicConfig(
    level=logging.DEBUG,
    handlers=[
        logging.StreamHandler(),
        logging.FileHandler("app.log"),
    ]
)

5 · Logger Objects

Always get a logger by name — typically __name__ so the hierarchy mirrors the package structure.

import logging

# Module-level logger (best practice)
logger = logging.getLogger(__name__)

# Explicit name
db_logger = logging.getLogger("myapp.db")
db_logger.setLevel(logging.WARNING)   # only WARNING+ from db
db_logger.propagate = False             # don't bubble up to root

# Root logger (use sparingly)
root = logging.getLogger()

# Check effective level
print(logger.getEffectiveLevel())       # 20 (INFO)
print(logger.isEnabledFor(logging.DEBUG)) # False

6 · Handlers

Handlers send LogRecords to a destination. Multiple handlers can be attached to one logger.

Built-in handlers

import logging
import logging.handlers

# Console
ch = logging.StreamHandler()            # sys.stderr by default
ch = logging.StreamHandler(sys.stdout)

# File
fh = logging.FileHandler("app.log", encoding="utf-8")

# Rotating by size
rh = logging.handlers.RotatingFileHandler(
    "app.log", maxBytes=5_000_000, backupCount=5
)

# Rotating by time
th = logging.handlers.TimedRotatingFileHandler(
    "app.log", when="midnight", backupCount=30, utc=True
)

# Syslog
sh = logging.handlers.SysLogHandler(address="/dev/log")

# HTTP POST
hh = logging.handlers.HTTPHandler("logs.example.com", "/log", method="POST")

# Queue (async, multiprocess-safe)
qh = logging.handlers.QueueHandler(queue)

# Memory (testing)
mh = logging.handlers.MemoryHandler(capacity=1000, target=fh)

# Attach to logger
logger.addHandler(ch)
logger.addHandler(fh)
logger.removeHandler(fh)

Formatters convert a LogRecord to a string. Attach one per handler.

fmt = logging.Formatter(

8 · Filters

Filters can be attached to loggers or handlers for fine-grained control beyond level checking.

# Class-based filter
class HealthCheckFilter(logging.Filter):
    """Suppress noisy health-check endpoint log lines."""
    def filter(self, record: logging.LogRecord) -> bool:
        return "GET /health" not in record.getMessage()

handler.addFilter(HealthCheckFilter())

# Function-based filter (Python ≥ 3.2 accepts callables)
handler.addFilter(lambda r: r.levelno != logging.DEBUG)

# Filter that injects context into every record
class RequestIdFilter(logging.Filter):
    def filter(self, record):
        record.request_id = get_current_request_id()   # thread-local / contextvars
        return True

9 · File Logging – Complete Example

import logging
import sys

def setup_logging(log_file="app.log", console_level=logging.INFO, file_level=logging.DEBUG):
    logger = logging.getLogger()          # root logger
    logger.setLevel(logging.DEBUG)         # must be ≤ lowest handler level

    fmt = logging.Formatter(
        "%(asctime)s [%(levelname)-8s] %(name)s — %(message)s",
        datefmt="%Y-%m-%dT%H:%M:%S",
    )

    # Console handler
    ch = logging.StreamHandler(sys.stdout)
    ch.setLevel(console_level)
    ch.setFormatter(fmt)

    # File handler
    fh = logging.FileHandler(log_file, encoding="utf-8")
    fh.setLevel(file_level)
    fh.setFormatter(fmt)

    logger.addHandler(ch)
    logger.addHandler(fh)
    return logger


logger = setup_logging()
logger.info("App started")

11 · dictConfig (recommended for apps)

logging.config.dictConfig() is the modern, version-controlled way to configure logging. The dict can be loaded from JSON, YAML, or a Python file.

import logging.config

LOGGING = {
    "version": 1,
    "disable_existing_loggers": False,
    "formatters": {
        "standard": {
            "format": "%(asctime)s [%(levelname)s] %(name)s — %(message)s",
            "datefmt": "%Y-%m-%dT%H:%M:%S",
        },
        "json": {
            "()": "myapp.logging.JsonFormatter",   # custom class
        },
    },
    "filters": {
        "no_health": {"()": "myapp.logging.HealthCheckFilter"},
    },
    "handlers": {
        "console": {
            "class": "logging.StreamHandler",
            "level": "INFO",
            "formatter": "standard",
            "stream": "ext://sys.stdout",
        },
        "file": {
            "class": "logging.handlers.RotatingFileHandler",
            "level": "DEBUG",
            "formatter": "standard",
            "filename": "logs/app.log",
            "maxBytes": 10485760,
            "backupCount": 5,
            "encoding": "utf-8",
            "filters": ["no_health"],
        },
    },
    "loggers": {
        "myapp": {"level": "DEBUG", "handlers": ["console", "file"], "propagate": False},
        "myapp.db": {"level": "WARNING"},  # inherits handlers from "myapp"

    "root": {"level": "WARNING", "handlers": ["console"]},

13 · Exceptions & Tracebacks

# Automatically include traceback with exc_info=True
try:
    risky_operation()
except ValueError as e:
    logger.error("Validation failed", exc_info=True)
    # shorthand — identical result:
    logger.exception("Validation failed")    # always logs at ERROR + traceback

# Include stack trace at any level
logger.debug("Debug checkpoint", stack_info=True)

# Re-raise after logging
try:
    connect()
except ConnectionError:
    logger.exception("Could not connect")
    raise                                     # re-raises the original exception
Use logger.exception() only inside except blocks — it reads sys.exc_info(). Outside an except block it logs NoneType: None as the exception.

14 · Structured / JSON Logging

Structured logs (machine-parseable JSON) are essential for log aggregation tools (ELK, Splunk, CloudWatch, etc.).

import json
import logging
import traceback

class JsonFormatter(logging.Formatter):
    def format(self, record: logging.LogRecord) -> str:
        log_obj = {
            "timestamp": self.formatTime(record, self.datefmt),
            "level":     record.levelname,
            "logger":   record.name,
            "message":  record.getMessage(),
            "module":   record.module,
            "function": record.funcName,
            "line":     record.lineno,
        }
        if record.exc_info:
            log_obj["exception"] = traceback.format_exception(*record.exc_info)
        # Attach any extra keys passed via logger.info(…, extra={…})
        for key, val in record.__dict__.items():
            if key not in logging.LogRecord("",0,"",0,"",(),None).__dict__:
                log_obj[key] = val
        return json.dumps(log_obj)

# Attach to handler:
handler.setFormatter(JsonFormatter())
python-json-logger (pip install python-json-logger) is the most popular drop-in JSON formatter if you don't want to write your own.

15 · Extra Fields & LoggerAdapter

# ── extra= dict ─────────────────────────────────────────────────
logger.info("Request received", extra={"request_id": "abc-123", "user": "alice"})
# Fields are accessible in Formatter as %(request_id)s, %(user)s

# ── LoggerAdapter (wraps a logger, auto-injects context) ─────────
class ContextAdapter(logging.LoggerAdapter):
    def process(self, msg, kwargs):
        kwargs.setdefault("extra", {}).update(self.extra)
        return msg, kwargs

base_logger = logging.getLogger(__name__)
logger = ContextAdapter(base_logger, {"service": "payment-svc", "env": "prod"})
logger.info("Payment processed")   # always includes service + env

# ── contextvars (Python ≥ 3.7) — thread/async safe ──────────────
from contextvars import ContextVar
_request_id: ContextVar[str] = ContextVar("request_id", default="-")

class ContextVarFilter(logging.Filter):
    def filter(self, record):
        record.request_id = _request_id.get()
        return True

16 · Multiprocess-Safe Logging

FileHandler is not safe when multiple processes write to the same file. Use QueueHandler + QueueListener instead.

import logging
import logging.handlers
from multiprocessing import Queue

# ── In the main process ──────────────────────────────────────────
log_queue = Queue(maxsize=-1)

# The listener runs in its own thread and writes to the real handler
file_handler = logging.FileHandler("app.log")
file_handler.setFormatter(logging.Formatter("%(asctime)s %(message)s"))

listener = logging.handlers.QueueListener(log_queue, file_handler, respect_handler_level=True)
listener.start()

# ── In each worker process / thread ─────────────────────────────
qh = logging.handlers.QueueHandler(log_queue)
worker_logger = logging.getLogger()
worker_logger.addHandler(qh)
worker_logger.setLevel(logging.DEBUG)

# ── Shutdown ─────────────────────────────────────────────────────
listener.stop()

18 · Best Practices

  • Always use logger = logging.getLogger(__name__) in every module — never the root logger in library code.
  • Set the root logger level once in application entry-point (main.py / app.py); libraries should not call basicConfig.
  • Use lazy % formatting (logger.debug("v=%s", val)) — avoids building the string when the level is filtered out.
  • Use logger.exception() (not logger.error()) inside except blocks to capture tracebacks automatically.
  • Prefer dictConfig over basicConfig for anything beyond a single script.
  • Set disable_existing_loggers: false in dictConfig to avoid silencing third-party library loggers.
  • In libraries: only add a NullHandler to your package logger — let the application configure the real handlers.
  • Use QueueHandler + QueueListener for async / multiprocess applications.
  • Emit structured (JSON) logs in production to enable log aggregation and querying.
  • Never log secrets, PII, or tokens. Scrub or mask sensitive fields in a custom Filter.
  • Include a correlation / request ID in every log record (via Filter + contextvars) for distributed tracing.
Library authors: the correct way to "be logging-friendly" is: 
import logging
logging.getLogger(__name__).addHandler(logging.NullHandler())
This prevents "No handlers could be found" warnings and lets the application control output.

19 · Real-World Use Cases

The following examples show complete, production-ready logging patterns for the most common application scenarios.

19.1 · REST API — per-request correlation ID

Inject a unique request ID into every log line so you can trace a single HTTP request across dozens of log entries in Kibana / Splunk.

# middleware.py  (Flask / FastAPI pattern — same idea)
import uuid, logging
from contextvars import ContextVar

_req_id: ContextVar[str] = ContextVar("request_id", default="-")
logger = logging.getLogger(__name__)

class RequestIdFilter(logging.Filter):
    def filter(self, record):
        record.request_id = _req_id.get()
        return True

# Attach filter once at startup
for handler in logging.getLogger().handlers:
    handler.addFilter(RequestIdFilter())

# Format string includes %(request_id)s
fmt = logging.Formatter(
    "%(asctime)s [%(levelname)s] %(name)s req=%(request_id)s — %(message)s"
)

# ── Middleware (called on every request) ──────────────────────────
def set_request_id(request_id: str = None):
    rid = request_id or str(uuid.uuid4())
    _req_id.set(rid)
    return rid

# Usage in a FastAPI route:
from fastapi import Request

async def request_logging_middleware(request: Request, call_next):
    rid = set_request_id(request.headers.get("X-Request-ID"))
    logger.info("START %s %s", request.method, request.url.path)
    response = await call_next(request)
    logger.info("END   %s %s status=%d", request.method, request.url.path, response.status_code)
    response.headers["X-Request-ID"] = rid
    return response
Every log line from every function called during that request now automatically carries req=3fa2… — no need to pass a logger or ID through the call stack.

19.2 · Database query auditing

Log every slow query above a threshold and all write operations (INSERT/UPDATE/DELETE) to a dedicated audit log, separate from the application log.

import logging, time, functools

audit_logger = logging.getLogger("audit.db")
perf_logger  = logging.getLogger("perf.db")

SLOW_QUERY_THRESHOLD_MS = 200
WRITE_VERBS = ("INSERT", "UPDATE", "DELETE", "MERGE")

def execute_query(conn, sql: str, params=()):
    start = time.perf_counter()
    try:
        cursor = conn.execute(sql, params)
        elapsed_ms = (time.perf_counter() - start) * 1000

        # Slow query warning
        if elapsed_ms > SLOW_QUERY_THRESHOLD_MS:
            perf_logger.warning(
                "SLOW QUERY %.1f ms | %s | params=%s",
                elapsed_ms, sql[:120], params
            )
        else:
            perf_logger.debug("Query %.1f ms | %s", elapsed_ms, sql[:80])

        # Write-operation audit trail
        verb = sql.strip().split()[0].upper()
        if verb in WRITE_VERBS:
            audit_logger.info(
                "%s rows_affected=%d sql=%s params=%s",
                verb, cursor.rowcount, sql[:200], params
            )
        return cursor
    except Exception:
        perf_logger.exception("Query FAILED: %s", sql[:200])
        raise

# Separate rotating file just for audit logs:
audit_handler = logging.handlers.TimedRotatingFileHandler(
    "logs/audit.log", when="midnight", backupCount=90, utc=True
)
audit_logger.addHandler(audit_handler)
audit_logger.propagate = False   # keep audit entries out of the main log

19.3 · Background job / ETL pipeline

Track each job run with start/end timing, record counts, and errors without stopping the pipeline.

import logging
from dataclasses import dataclass, field
from typing import List

logger = logging.getLogger("etl.pipeline")

@dataclass
class JobStats:
    processed: int = 0
    skipped:   int = 0
    errors:    int = 0
    failures:  List[str] = field(default_factory=list)

def run_pipeline(job_name: str, records):
    stats = JobStats()
    logger.info("[%s] Pipeline STARTED — %d records to process", job_name, len(records))

    for i, record in enumerate(records):
        try:
            if not is_valid(record):
                logger.debug("[%s] Record #%d skipped — validation failed: %s", job_name, i, record)
                stats.skipped += 1
                continue
            transform_and_load(record)
            stats.processed += 1
        except Exception as e:
            logger.error("[%s] Record #%d FAILED: %s", job_name, i, e)
            stats.errors += 1
            stats.failures.append(str(record))

    level = logging.WARNING if stats.errors else logging.INFO
    logger.log(level,
        "[%s] Pipeline FINISHED — processed=%d skipped=%d errors=%d",
        job_name, stats.processed, stats.skipped, stats.errors
    )
    if stats.failures:
        logger.warning("[%s] Failed records: %s", job_name, stats.failures[:10])
    return stats

19.4 · Authentication & security event logging

Keep security events in a dedicated log stream (separate from application logs) for SIEM ingestion and compliance.

import logging

sec_logger = logging.getLogger("security")
# Route to a separate file / SIEM handler — never propagate
sec_logger.propagate = False

# ── Login attempt ──────────────────────────────────────────────────
def log_login_attempt(username: str, ip: str, success: bool, reason: str = ""):
    event = "LOGIN_SUCCESS" if success else "LOGIN_FAILURE"
    level = logging.INFO if success else logging.WARNING
    sec_logger.log(level, "%s user=%s ip=%s reason=%s", event, username, ip, reason)

# ── Brute-force detection ──────────────────────────────────────────
def log_account_locked(username: str, ip: str, attempt_count: int):
    sec_logger.warning(
        "ACCOUNT_LOCKED user=%s ip=%s attempts=%d", username, ip, attempt_count
    )

# ── Privilege escalation ───────────────────────────────────────────
def log_privilege_change(actor: str, target: str, old_role: str, new_role: str):
    sec_logger.info(
        "PRIVILEGE_CHANGE actor=%s target=%s %s→%s", actor, target, old_role, new_role
    )

# ── Suspicious activity ────────────────────────────────────────────
def log_suspicious_request(ip: str, path: str, reason: str):
    sec_logger.error(
        "SUSPICIOUS_REQUEST ip=%s path=%s reason=%s", ip, path, reason
    )

# Example output lines (JSON formatter attached to sec_logger):
# {"event":"LOGIN_FAILURE","user":"alice","ip":"203.0.113.5","reason":"bad_password"}
# {"event":"ACCOUNT_LOCKED","user":"alice","ip":"203.0.113.5","attempts":5}
Never log plain-text passwords, tokens, or full credit card numbers. Log only the username, IP, event type, and a sanitised reason code.

19.5 · Microservice with distributed tracing (OpenTelemetry)

Bridge Python logging with OpenTelemetry so every log record carries the active trace_id and span_id — enabling jump-from-log-to-trace in tools like Jaeger or Grafana Tempo.

# pip install opentelemetry-sdk opentelemetry-instrumentation-logging
from opentelemetry import trace

import logging

# Automatically injects trace_id + span_id into every LogRecord

logger = logging.getLogger("payment-service")

def process_payment(order_id: str, amount: float):
    tracer = trace.get_tracer(__name__)
    with tracer.start_as_current_span("process_payment") as span:
        span.set_attribute("order.id", order_id)
        logger.info("Processing payment order=%s amount=%.2f", order_id, amount)
        try:

            return result
        except PaymentDeclinedError as e:

19.6 · Celery / async task worker

Log task lifecycle (received → started → succeeded / failed / retried) with task ID so you can track individual jobs through the queue.

from celery import Celery, signals
import logging

app = Celery("tasks")
logger = logging.getLogger("celery.tasks")

# ── Per-task logging with task ID ─────────────────────────────────
@app.task(bind=True, max_retries=3)
def send_notification(self, user_id: int, message: str):
    task_id = self.request.id
    logger.info("[%s] Sending notification to user=%d", task_id, user_id)
    try:
        deliver(user_id, message)
        logger.info("[%s] Notification delivered user=%d", task_id, user_id)
    except TransientError as exc:
        logger.warning("[%s] Transient error — retrying (%d/%d): %s",
                        task_id, self.request.retries + 1, self.max_retries, exc)
        raise self.retry(exc=exc, countdown=2 ** self.request.retries)
    except Exception:
        logger.exception("[%s] Notification FAILED user=%d", task_id, user_id)
        raise

# ── Worker-level signals for operational monitoring ───────────────
@signals.worker_ready.connect
def on_worker_ready(**kwargs):
    logger.info("Celery worker READY")

@signals.worker_shutdown.connect
def on_worker_shutdown(**kwargs):
    logger.info("Celery worker SHUTDOWN")

19.7 · CLI application with --verbose / --quiet flags

Map user-facing -v / -q flags to logging levels so power users get debug output while normal runs stay quiet.

import argparse, logging, sys

def setup_cli_logging(verbosity: int):
    # 0 = WARNING, 1 = INFO (-v), 2 = DEBUG (-vv)
    level = {
        0: logging.WARNING,
        1: logging.INFO,
        2: logging.DEBUG,
    }.get(verbosity, logging.DEBUG)

    logging.basicConfig(
        level=level,
        format="%(levelname)s: %(message)s" if verbosity < 2 else

19.10 · Alerting on ERROR logs (Slack / PagerDuty webhook)

Attach a custom handler that fires a webhook whenever a CRITICAL or ERROR event is logged — no external daemon required.

import logging, urllib.request, json

class SlackAlertHandler(logging.Handler):
    def __init__(self, webhook_url: str, level=logging.ERROR):
        super().__init__(level)
        self.webhook_url = webhook_url

    def emit(self, record: logging.LogRecord):
        try:
            text = ">:rotating_light: *{}* `{}` — {}".format(
                record.levelname, record.name, self.format(record)
            )
            payload = json.dumps({"text": text}).encode()
            req = urllib.request.Request(
                self.webhook_url,
                data=payload,
                headers={"Content-Type": "application/json"},
                method="POST",
            )
            with urllib.request.urlopen(req, timeout=5):
                pass
        except Exception:
            self.handleError(record)   # never propagate handler exceptions

# Wire it up — only fires for ERROR and above
slack_handler = SlackAlertHandler("https://hooks.slack.com/services/T…/B…/…")
logging.getLogger().addHandler(slack_handler)

20.2 · Dataset loading & preprocessing pipeline

Track record counts, drop reasons, and timing at each stage so data quality issues are caught before training starts.

import logging, time

data_logger = logging.getLogger("ml.data")

class DataPipelineLogger:
    def __init__(self, dataset_name: str):
        self.name = dataset_name
        self.stage_counts = {}

    def stage(self, stage_name: str, input_count: int, output_count: int, elapsed_s: float):
        dropped  = input_count - output_count
        drop_pct = (dropped / input_count * 100) if input_count else 0
        level = logging.WARNING if drop_pct > 10 else logging.INFO
        data_logger.log(level,
            "[%s] stage=%s in=%d out=%d dropped=%d (%.1f%%) time=%.2fs",
            self.name, stage_name, input_count, output_count, dropped, drop_pct, elapsed_s
        )
        if drop_pct > 10:
            data_logger.warning(
                "[%s] HIGH DROP RATE at '%s' — check filtering logic", self.name, stage_name
            )
        self.stage_counts[stage_name] = output_count

    def summary(self):
        data_logger.info("[%s] Pipeline summary: %s", self.name, self.stage_counts)

# Usage
plog = DataPipelineLogger("ImageNet-mini")

20.3 · Gradient & weight anomaly detection during training

Log NaN / Inf gradients and exploding weight norms before they silently corrupt a multi-hour training run.

import logging
import torch

grad_logger = logging.getLogger("ml.gradients")

def check_gradients(model, epoch: int, step: int):
    total_norm = 0.0
    has_nan    = False
    for name, param in model.named_parameters():
        if param.grad is None:
            continue
        grad = param.grad.data
        if torch.isnan(grad).any() or torch.isinf(grad).any():
            grad_logger.error(
                "NaN/Inf gradient epoch=%d step=%d param=%s", epoch, step, name
            )
            has_nan = True
        total_norm += grad.norm().item() ** 2
    total_norm = total_norm ** 0.5
    if total_norm > 100:
        grad_logger.warning("Exploding gradient epoch=%d step=%d norm=%.2f", epoch, step, total_norm)
    elif step % 100 == 0:
        grad_logger.debug("Gradient norm epoch=%d step=%d norm=%.4f", epoch, step, total_norm)
    return has_nan

# Call after loss.backward() and before optimizer.step()
if check_gradients(model, epoch, step):
    grad_logger.critical("Aborting: NaN gradients at epoch=%d step=%d", epoch, step)
    break

Log prompt tokens, completion tokens, cost estimates, and latency for every LLM call to track spend and detect regressions.

import logging, time
from openai import OpenAI

COST_PER_1K = {"gpt-4o": (0.005, 0.015), "gpt-4o-mini": (0.00015, 0.0006)}

def chat_completion(client: OpenAI, model: str, messages: list, **kwargs) -> str:

    latency_ms = (time.perf_counter() - t0) * 1000

    usage  = response.usage
    p_tok  = usage.prompt_tokens
    c_tok  = usage.completion_tokens

        llm_logger.warning("TOKEN_LIMIT_HIT model=%s — response may be truncated", model)

20.6 · RAG pipeline — retrieval & generation tracing

For Retrieval-Augmented Generation systems, log query embedding time, number of chunks retrieved, rerank scores, and final generation latency to diagnose retrieval quality issues.

import logging, time

rag_logger = logging.getLogger("ai.rag")

def rag_query(question: str, retriever, generator, session_id: str) -> str:
    rag_logger.info("RAG_START session=%s question_len=%d", session_id, len(question))

    # ── Retrieval ──────────────────────────────────────────────────
    t0     = time.perf_counter()
    chunks = retriever.retrieve(question)
    ret_ms = (time.perf_counter() - t0) * 1000
    scores = [c.score for c in chunks]

    rag_logger.info(
        "RAG_RETRIEVE session=%s chunks=%d top_score=%.3f min_score=%.3f latency_ms=%.0f",
        session_id, len(chunks),
        max(scores, default=0), min(scores, default=0), ret_ms
    )
    if not chunks:
        rag_logger.warning("RAG_NO_CHUNKS session=%s — generation may hallucinate", session_id)
    elif max(scores) < 0.5:
        rag_logger.warning(
            "RAG_LOW_RELEVANCE session=%s top_score=%.3f — consider query rewriting",
            session_id, max(scores)
        )

    # ── Generation ─────────────────────────────────────────────────
    t1     = time.perf_counter()
    answer = generator.generate(question, chunks)
    gen_ms = (time.perf_counter() - t1) * 1000

    rag_logger.info(
        "RAG_GENERATE session=%s answer_len=%d gen_ms=%.0f",
        session_id, len(answer), gen_ms
    )
    return answer

20.7 · Model drift & data quality monitoring

Compare live inference distributions against training baselines and log drift alerts before model quality degrades silently.

import logging
from scipy.stats import ks_2samp
import numpy as np

drift_logger      = logging.getLogger("ml.drift")
DRIFT_P_THRESHOLD = 0.05   # KS p-value below this → drift
MISSING_RATE_WARN = 0.02   # 2% missing values triggers warning

def check_feature_drift(feature_name: str, train_data: np.ndarray, live_data: np.ndarray):
    stat, p_value = ks_2samp(train_data, live_data)
    drift_logger.info(
        "DRIFT_CHECK feature=%s ks_stat=%.4f p_value=%.4f", feature_name, stat, p_value
    )
    if p_value < DRIFT_P_THRESHOLD:
        drift_logger.warning(
            "DRIFT_DETECTED feature=%s ks_stat=%.4f p=%.4f — retrain may be needed",
            feature_name, stat, p_value
        )

def check_missing_values(batch: np.ndarray, feature_names: list):
    for i, name in enumerate(feature_names):
        missing_rate = np.isnan(batch[:, i]).mean()
        if missing_rate > MISSING_RATE_WARN:
            drift_logger.warning(
                "HIGH_MISSING feature=%s rate=%.2f%%", name, missing_rate * 100
            )

def check_prediction_drift(model_version: str, baseline_preds, live_preds):
    shift = abs(np.mean(live_preds) - np.mean(baseline_preds))
    drift_logger.info(
        "PRED_DRIFT model=%s baseline_mean=%.4f live_mean=%.4f shift=%.4f",
        model_version, np.mean(baseline_preds), np.mean(live_preds), shift
    )
    if shift > 0.1:
        drift_logger.error(
            "PRED_DRIFT_HIGH model=%s shift=%.4f — investigate immediately",
            model_version, shift
        )

20.8 · AI agent tool-call tracing

When an LLM agent invokes tools (function calling / ReAct), log every tool call, its arguments, result, and elapsed time so you can replay and debug agent reasoning.

import logging, time, json
from functools import wraps

agent_logger = logging.getLogger("ai.agent")

def tool(func):
    """Decorator that logs every agent tool invocation."""
    @wraps(func)
    def wrapper(*args, **kwargs):
        call_id = str(id(wrapper))[-6:]
        agent_logger.info(
            "TOOL_CALL id=%s tool=%s args=%s",
            call_id, func.__name__,
            json.dumps({"args": args[1:], "kwargs": kwargs}, default=str)
        )
        t0 = time.perf_counter()
        try:
            result = func(*args, **kwargs)
            ms = (time.perf_counter() - t0) * 1000
            agent_logger.info(
                "TOOL_RESULT id=%s tool=%s latency_ms=%.0f result=%s",
                call_id, func.__name__, ms,
                json.dumps(result, default=str)[:200]   # truncate large results
            )
            return result
        except Exception as e:
            ms = (time.perf_counter() - t0) * 1000
            agent_logger.error(
                "TOOL_ERROR id=%s tool=%s latency_ms=%.0f error=%s",
                call_id, func.__name__, ms, e
            )
            raise
    return wrapper

# Decorate your agent tools:
@tool
def search_knowledge_base(self, query: str, top_k: int = 5):
    return self.retriever.search(query, top_k)

@tool
def execute_sql(self, query: str):
    return self.db.run(query)

# TOOL_CALL  id=abc123 tool=search_knowledge_base args={"args":["refund policy"],...}
# TOOL_RESULT id=abc123 tool=search_knowledge_base latency_ms=42 result=[{"chunk":"..."}]
Combine with a structured JSON formatter and ship agent logs to an LLM observability platform (LangSmith, Helicone, Arize) for chain-of-thought analytics.

import logging

21 · Other Important Logging Concepts

21.1 · Custom Log Levels

Add levels between the built-ins when you need finer granularity (e.g. VERBOSE between DEBUG and INFO, AUDIT above WARNING).

import logging

VERBOSE = 15   # between DEBUG (10) and INFO (20)
AUDIT   = 35   # between WARNING (30) and ERROR (40)

logging.addLevelName(VERBOSE, "VERBOSE")
logging.addLevelName(AUDIT,   "AUDIT")

def verbose(self, message, *args, **kwargs):
    if self.isEnabledFor(VERBOSE):
        self._log(VERBOSE, message, args, **kwargs)

def audit(self, message, *args, **kwargs):
    if self.isEnabledFor(AUDIT):
        self._log(AUDIT, message, args, **kwargs)

logging.Logger.verbose = verbose
logging.Logger.audit   = audit

logger = logging.getLogger(__name__)
logger.setLevel(VERBOSE)
logger.verbose("Detailed trace: x=%s", x)
logger.audit("User %s deleted record %s", user_id, record_id)
Custom levels are global — once registered with addLevelName they apply to the whole process. Avoid creating too many; prefer structured fields in log records instead.

21.2 · Logging in asyncio

The stdlib logging module is thread-safe but its handlers do blocking I/O. In async applications, offload writes to a background thread with QueueHandler so you never block the event loop.

import asyncio, logging, logging.handlers
from queue import SimpleQueue

def setup_async_logging():
    log_queue = SimpleQueue()

    file_handler = logging.handlers.RotatingFileHandler(
        "app.log", maxBytes=10_485_760, backupCount=5
    )
    file_handler.setFormatter(logging.Formatter(
        "%(asctime)s [%(levelname)s] %(name)s -- %(message)s"
    ))

    # Listener runs in a daemon thread — never blocks the event loop
    listener = logging.handlers.QueueListener(
        log_queue, file_handler, respect_handler_level=True
    )
    listener.start()

    root = logging.getLogger()
    root.addHandler(logging.handlers.QueueHandler(log_queue))
    root.setLevel(logging.DEBUG)
    return listener

# Inside an async handler — logger.info() returns instantly
logger = logging.getLogger(__name__)

async def handle_request(request_id: str):
    logger.info("req=%s started", request_id)
    await asyncio.sleep(0)
    logger.info("req=%s finished", request_id)

21.5 · Sensitive Data Masking

A Filter applied to the logger (not just a handler) scrubs secrets before they reach any destination.

import logging, re

class SensitiveDataFilter(logging.Filter):
    """Redact passwords, tokens, and credit-card-like numbers."""
    PATTERNS = [
        # key=value pairs
        (re.compile(r'(?i)(password|passwd|secret|token|api[_-]?key)=\S+'),
         lambda m: m.group(1) + "=***REDACTED***"),
        # 16-digit card numbers
        (re.compile(r'\b(\d{4}[\s-]?){3}\d{4}\b'),
         lambda m: "****-****-****-" + m.group()[-4:]),
        # Bearer tokens
        (re.compile(r'Bearer\s+[A-Za-z0-9\-._~+/]+'),
         lambda _: "Bearer ***REDACTED***"),
    ]

    def filter(self, record: logging.LogRecord) -> bool:
        msg = record.getMessage()
        for pattern, replacement in self.PATTERNS:
            msg = pattern.sub(replacement, msg)
        record.msg  = msg
        record.args = ()   # args already baked into msg
        return True

# Apply to the root logger — no handler ever sees raw secrets
logging.getLogger().addFilter(SensitiveDataFilter())
Apply the masking filter to the logger, not just a handler. A handler added later (e.g. a debug console) would otherwise bypass handler-level filters.

21.6 · Log Sampling & Rate Limiting

High-throughput services can emit millions of records per minute. Sampling keeps logs useful without drowning your storage.

import logging, time
from collections import defaultdict

class RateLimitFilter(logging.Filter):
    """Allow at most `rate` identical messages per `per` seconds.
    Appends suppression count when the window resets."""
    def __init__(self, rate: int = 5, per: float = 60.0):
        super().__init__()
        self.rate        = rate
        self.per         = per
        self._counts     = defaultdict(int)
        self._windows    = defaultdict(float)
        self._suppressed = defaultdict(int)

    def filter(self, record: logging.LogRecord) -> bool:
        key = (record.name, record.levelno, record.getMessage())
        now = time.monotonic()
        if now - self._windows[key] >= self.per:
            suppressed = self._suppressed[key]
            if suppressed:
                record.msg  = record.msg + " [+%d suppressed]" % suppressed
                record.args = ()
            self._counts[key]     = 1
            self._windows[key]    = now
            self._suppressed[key] = 0
            return True
        if self._counts[key] < self.rate:
            self._counts[key] += 1
            return True
        self._suppressed[key] += 1
        return False

# Max 3 identical messages per 10 seconds:
handler.addFilter(RateLimitFilter(rate=3, per=10.0))

21.7 · Temporary Level Change (Context Manager)

Temporarily lower a logger's level for a noisy library call, then restore it automatically — useful in tests and targeted debugging.

import logging
from contextlib import contextmanager

@contextmanager
def log_level(logger_name: str, level: int):
    """Temporarily set a logger to `level`, restore on exit."""
    logger    = logging.getLogger(logger_name)
    old_level = logger.level
    logger.setLevel(level)
    try:
        yield logger
    finally:
        logger.setLevel(old_level)

# Silence a chatty library during a known-noisy operation
with log_level("urllib3", logging.WARNING):
    make_many_http_requests()

# Enable verbose DEBUG output only for a specific block
with log_level("myapp.payment", logging.DEBUG):
    process_payment(order)

21.8 · logging.disable() and logging.shutdown()

import logging, atexit, os

# logging.disable(level) globally suppresses all records at or below
# the given level — faster than setting every logger individually.
if os.getenv("ENV") == "production":
    logging.disable(logging.DEBUG)    # suppress DEBUG everywhere

logging.disable(logging.NOTSET)      # re-enable everything

# logging.shutdown() flushes and closes all handlers.
# Called automatically at interpreter exit, but register it
# explicitly when using network / socket handlers that need
# guaranteed flushing before the process dies:
atexit.register(logging.shutdown)

21.9 · The lastResort Handler & Library Etiquette

When no handlers are configured on the root logger, Python 3.2+ uses a built-in _defaultLastResort handler that writes WARNING+ to sys.stderr. Understanding this prevents confusing "ghost" log lines.

# If root logger has no handlers and propagation reaches it,
# logging._defaultLastResort (StreamHandler at WARNING) catches it.
# This is why WARNING appears on stderr even with no basicConfig call.

# Correct library pattern -- add NullHandler in your package __init__.py
# so the lastResort never fires for your library's loggers:
import logging
logging.getLogger("mylib").addHandler(logging.NullHandler())

# Silence chatty third-party libraries in your application:
for noisy in ("boto3", "botocore", "urllib3", "httpx"):
    logging.getLogger(noisy).setLevel(logging.WARNING)

21.10 · Zero-Cost Logging Guards (Performance)

Logging calls in hot paths have measurable overhead even when filtered out, because argument evaluation happens before the level check.

import logging, json
logger = logging.getLogger(__name__)

# BAD: f-string always evaluated, even when DEBUG is off (~420 ns)
logger.debug(f"state={expensive_repr(obj)}")

# GOOD: % format — string only built if level passes (~180 ns)
logger.debug("state=%s", obj)

# BETTER for truly expensive calls: explicit guard (~45 ns when off)
if logger.isEnabledFor(logging.DEBUG):
    logger.debug("state=%s", expensive_repr(obj))

# PATTERN: Lazy wrapper -- __str__ only called when record is emitted
class Lazy:
    def __init__(self, fn): self.fn = fn
    def __str__(self):      return str(self.fn())

logger.debug("dump=%s", Lazy(lambda: json.dumps(big_dict)))
In tight loops (>10k calls/sec) the isEnabledFor() guard is the recommended pattern. The difference between % and f-strings is negligible in normal code but measurable in hot paths.

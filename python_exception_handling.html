1. Basic try / except / else / finally

The fundamental building block. Each clause has a distinct role:

pythontry:
    result = 10 / 0           # code that may raise

except ZeroDivisionError as e:
    print(f"Caught: {e}")      # handle the specific error

else:
    print(f"Result: {result}")   # runs ONLY when no exception occurred

finally:
    print("Always executes")    # cleanup — runs regardless

try

The block of code to monitor for exceptions. Execution stops at the first raised exception.

except

Handles a specific exception type. Multiple except clauses are evaluated top-to-bottom.

else

Runs only when the try block completed without raising. Useful to keep "success" logic separate.

finally

Always executes — even if the handler re-raises or a return is hit. Use for resource cleanup.
Note: else and finally are optional. A bare try/except with no else or finally is perfectly valid.

2. Exception Hierarchy

All exceptions derive from BaseException. Catching a parent class also catches all its children.

BaseException ├── SystemExit # raised by sys.exit() ├── KeyboardInterrupt # Ctrl+C ├── GeneratorExit # generator.close() └── Exception # base of all normal errors ├── ArithmeticError │ ├── ZeroDivisionError │ ├── OverflowError │ └── FloatingPointError ├── LookupError │ ├── IndexError │ └── KeyError ├── OSError (IOError, EnvironmentError) │ ├── FileNotFoundError │ ├── PermissionError │ ├── TimeoutError │ └── ConnectionError → BrokenPipeError, ConnectionResetError … ├── AttributeError ├── ImportError → ModuleNotFoundError ├── NameError → UnboundLocalError ├── TypeError ├── ValueError → UnicodeError ├── RuntimeError → RecursionError, NotImplementedError ├── StopIteration ├── MemoryError └── Warning → DeprecationWarning, UserWarning …
Caution: Catching Exception will NOT catch SystemExit, KeyboardInterrupt, or GeneratorExit — you need BaseException for those. In practice, almost never catch BaseException directly.

3. Catching Multiple Exceptions

Separate handlers

pythontry:
    data = process(value)
except ValueError:
    print("Bad value")
except TypeError:
    print("Wrong type")
except Exception as e:
    print(f"Unexpected: {e}")   # catch-all fallback

Grouped exceptions in a single handler

pythonexcept (ValueError, TypeError, KeyError) as e:
    print(f"Input error: {e}")

Re-raising inside a handler

pythonexcept Exception:
    log_error()
    raise           # bare raise — re-raises the original exception unchanged
Order matters: Handlers are tried top-to-bottom. Put specific exceptions before general ones, otherwise the general handler will shadow them.

Basic custom exception

Hierarchy with extra context

pythonclass AppError(Exception):
    """Base application error."""

class ValidationError(AppError):
    def __init__(self, field: str, message: str) -> None:

        super().__init__(f"{field}: {message}")

        self.id_ = id_

Usage

pythontry:

Pattern: Always create a base AppError(Exception) for your package. Users can then catch all your exceptions with a single type, or specific ones as needed.

7. Context Managers

with statements guarantee cleanup whether or not an exception occurs — the idiomatic replacement for try/finally.

Using built-in context managers

pythonwith open("data.txt", "r") as f:
    content = f.read()   # file is closed even if read() raises

# Multiple resources in one statement (Python 3.10+ parenthesised form)
with (
    open("input.txt") as src,
    open("output.txt", "w") as dst,
):
    dst.write(src.read())

Class-based context manager

pythonclass DatabaseConnection:
    def __enter__(self):
        self.conn = connect_db()
        return self.conn       # value bound to 'as' variable

    def __exit__(self, exc_type, exc_val, exc_tb):
        self.conn.close()
        # Return True  → suppress the exception
        # Return False → propagate the exception (default)
        return False

with DatabaseConnection() as conn:
    conn.execute("SELECT 1")

__exit__ parameters

ParameterValue when no exceptionValue when exception raised
exc_typeNoneException class (e.g. ValueError)
exc_valNoneException instance
exc_tbNoneTraceback object
return valueignoredTrue suppresses, False/None propagates

8. contextlib Utilities

@contextmanager — generator-based

pythonfrom contextlib import contextmanager

@contextmanager
def managed_transaction(conn):
    try:
        yield conn          # code inside 'with' block runs here
        conn.commit()
    except Exception:
        conn.rollback()
        raise
    finally:
        conn.close()

with managed_transaction(get_conn()) as conn:
    conn.execute("INSERT ...")

suppress — silently ignore specific exceptions

import os

    os.remove("tmp_file.txt")   # no error if file doesn't exist

ExitStack — dynamic resource management

pythonfrom contextlib import ExitStack

with ExitStack() as stack:
    files = [stack.enter_context(open(f)) for f in filenames]
    # all files are closed when the block exits, even if one open() fails

asynccontextmanager (async/await)

pythonfrom contextlib import asynccontextmanager

@asynccontextmanager
async def async_resource():
    resource = await acquire()
    try:
        yield resource
    finally:
        await resource.release()

async with async_resource() as r:
    await r.do_work()

9. Exception Groups (Python 3.11+)

Handle multiple simultaneous exceptions — primarily from concurrent code (e.g. asyncio.TaskGroup).

Raising an ExceptionGroup

pythonraise ExceptionGroup(
    "multiple errors",
    [ValueError("bad val"), TypeError("bad type"), KeyError("missing")]
)

Handling with except*

pythontry:
    async with asyncio.TaskGroup() as tg:
        tg.create_task(task_a())
        tg.create_task(task_b())
except* ValueError as eg:
    for exc in eg.exceptions:
        print(f"ValueError: {exc}")
except* TypeError as eg:
    print(f"{len(eg.exceptions)} TypeErrors occurred")
Note: except* and except cannot be mixed in the same try block. except* always receives an ExceptionGroup even if only one matching exception exists.

11. Common Patterns

Retry with backoff

pythonimport time

def retry(func, retries=3, delay=1.0, backoff=2.0):
    last_exc = None
    for attempt in range(retries):
        try:
            return func()
        except Exception as e:
            last_exc = e
            time.sleep(delay * (backoff ** attempt))
    raise last_exc

Fallback / default value

pythondef safe_parse_int(value, default=0):
    try:
        return int(value)
    except (ValueError, TypeError):
        return default

Guard clause / validate at boundary

pythondef process_order(order_id: int) -> Order:
    if not isinstance(order_id, int):
        raise TypeError(f"order_id must be int, got {type(order_id).__name__}")
    if order_id <= 0:
        raise ValueError("order_id must be positive")
    # ... rest of logic

Exception translation (layer boundary)

python# Translate low-level errors to domain errors at service boundaries
def get_user(user_id: int) -> User:
    try:
        return db.query(f"SELECT * FROM users WHERE id={user_id}")
    except DatabaseConnectionError as e:
        raise ServiceUnavailableError("User service unavailable") from e
    except RecordNotFoundError as e:
        raise NotFoundError("User", user_id) from e

Async exception handling

pythonimport asyncio

async def fetch_data(url: str) -> dict:
    try:
        async with session.get(url) as resp:
            resp.raise_for_status()
            return await resp.json()
    except asyncio.TimeoutError:

    except Exception as e:

12. Best Practices & Anti-patterns

✓ Do

Catch specific exception types, not broad ones.
Use raise ... from e to preserve the cause.
Put cleanup code in finally or use with.
Define a base error class per library/package.
Log with logger.exception() to capture the traceback.
Validate inputs at system/API boundaries.
Re-raise if you can't meaningfully handle an exception.

✗ Avoid

Bare except: — catches everything including KeyboardInterrupt.
Silently swallowing exceptions: except Exception: pass.
Using exceptions for normal control flow (EAFP has limits).
Accessing exception variable e after the except block — it's deleted.
Catching Exception in library code and not re-raising.
Raising bare Exception — always use a specific type.
Over-catching at low levels, hiding bugs from upper layers.

Variable scoping gotcha

pythontry:
    raise ValueError("oops")
except ValueError as e:
    err = str(e)   # save before the block ends!

# 'e' is deleted here — Python does this to break reference cycles
print(err)        # OK — we saved it in 'err'
# print(e)        # NameError!

class ManagedFile:
    def __init__(self, path: str, mode: str = "r"):
        self.path = path
        self.mode = mode

    def __enter__(self):
        print(f"[ManagedFile] Opening {self.path}")

Program 5 — Retry Decorator

A production-ready retry decorator with exponential backoff and selective exception types.

python# program5_retry_decorator.py
import time
import functools
import random

def retry(exceptions, retries=3, delay=0.5, backoff=2.0):
    """Retry decorator with exponential backoff.

    Args:
        exceptions: exception type or tuple of types to retry on
        retries:    maximum number of attempts
        delay:      initial wait in seconds
        backoff:    multiplier applied to delay after each failure
    """
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            wait = delay
            for attempt in range(1, retries + 1):
                try:
                    return func(*args, **kwargs)
                except exceptions as e:
                    if attempt == retries:
                        print(f"  [retry] All {retries} attempts failed. Raising.")
                        raise
                    print(f"  [retry] Attempt {attempt} failed: {e}. Retrying in {wait:.1f}s")
                    time.sleep(wait)
                    wait *= backoff
        return wrapper
    return decorator


# Simulated flaky service
_call_count = 0

@retry((ConnectionError, TimeoutError), retries=4, delay=0.1)
def flaky_api_call():
    global _call_count
    _call_count += 1
    print(f"  [api] attempt #{_call_count}")
    if _call_count < 3:
        raise ConnectionError("service unreachable")
    return "{'status': 'ok'}"


print("--- Retrying until success ---")
result = flaky_api_call()
print(f"  Result: {result}")

# Reset and test exhaustion
_call_count = 0

@retry(ConnectionError, retries=3, delay=0.1)
def always_fails():
    global _call_count
    _call_count += 1
    raise ConnectionError("always down")

    always_fails()
except ConnectionError as e:
    print(f"  Final error caught: {e}")

Program 6 — File Processing with Robust Error Handling

Reads a CSV-like file, validates each row, collects all errors, and reports a summary.

python# program6_file_processing.py
import io

# Simulated file content (use open() for real files)
CSV_DATA = """\
name,age,email
Alice,30,alice@example.com
Bob,abc,bob@example.com
Charlie,,charlie@example.com
Diana,25,not-an-email
Eve,17,eve@example.com
"""

class RowValidationError(Exception):
    def __init__(self, row: int, field: str, msg: str):
        self.row = row
        self.field = field
        super().__init__(f"Row {row}, field '{field}': {msg}")


def validate_row(row_num: int, name: str, age: str, email: str):
    errors = []
    if not name.strip():
        errors.append(RowValidationError(row_num, "name", "cannot be empty"))
    try:
        age_val = int(age)
        if age_val < 18:
            errors.append(RowValidationError(row_num, "age", f"must be >= 18, got {age_val}"))
    except ValueError:
        errors.append(RowValidationError(row_num, "age", f"must be integer, got '{age}'"))
    if "@" not in email or "." not in email.split("@")[-1]:
        errors.append(RowValidationError(row_num, "email", f"invalid format '{email}'"))
    return errors


def process_csv(data: str):
    valid_rows = []
    all_errors = []

    reader = io.StringIO(data)
    headers = reader.readline().strip().split(",")  # skip header

    for row_num, line in enumerate(reader, start=2):
        line = line.strip()
        if not line:
            continue
        try:
            parts = line.split(",")
            if len(parts) != 3:
                raise ValueError(f"expected 3 columns, got {len(parts)}")
            name, age, email = parts
        except ValueError as e:
            all_errors.append(RowValidationError(row_num, "row", str(e)))
            continue

        errors = validate_row(row_num, name, age, email)
        if errors:
            all_errors.extend(errors)
        else:
            valid_rows.append({"name": name, "age": int(age), "email": email})

    return valid_rows, all_errors


valid, errors = process_csv(CSV_DATA)

print(f"Valid rows ({len(valid)}):")
for r in valid:
    print(f"  {r}")

print(f"\nErrors ({len(errors)}):")
for e in errors:
    print(f"  {e}")

async def validate_all(items):
    """Validate all items and raise all errors at once."""
    errors = []
    for item in items:
        if not isinstance(item["age"], int):
            errors.append(TypeError(f"age must be int for {item['name']}"))
        elif item["age"] < 0:
            errors.append(ValueError(f"negative age for {item['name']}"))
    if errors:
        raise ExceptionGroup("validation errors", errors)
    return items

    dataset = [
        {"name": "Alice", "age": 30},
        {"name": "Bob",   "age": "old"},   # TypeError

    except* TypeError as eg:

    except* ValueError as eg:
        print(f"Value errors ({len(eg.exceptions)}):")
        for e in eg.exceptions:
            print(f"  {e}")

    # TaskGroup: all task errors collected automatically
    print("\n--- asyncio.TaskGroup ---")

        if fail:

            tg.create_task(task("A"))
            tg.create_task(task("B", fail=True))

    except* RuntimeError as eg:
        print(f"Caught {len(eg.exceptions)} RuntimeErrors from TaskGroup:")
        for e in eg.exceptions:
            print(f"  {e}")

import logging
from contextlib import contextmanager

logging.basicConfig(
    level=logging.INFO,
    format="%(levelname)-8s %(message)s"
)
logger = logging.getLogger("bank")


# --- Exceptions ---
class BankError(Exception):         pass
class InsufficientFundsError(BankError):
    def __init__(self, balance, amount):
        super().__init__(f"Cannot withdraw {amount:.2f}, balance is {balance:.2f}")
        self.balance = balance
        self.amount = amount

class NegativeAmountError(BankError):
    def __init__(self, amount):
        super().__init__(f"Amount must be positive, got {amount}")

class FrozenAccountError(BankError):
    def __init__(self, account_id):
        super().__init__(f"Account {account_id} is frozen")


# --- Account ---
class BankAccount:
    def __init__(self, account_id: str, balance: float = 0.0):
        self.account_id = account_id
        self._balance = balance
        self._frozen = False

    @property
    def balance(self): return self._balance

    def _check_frozen(self):
        if self._frozen:
            raise FrozenAccountError(self.account_id)

    def deposit(self, amount: float):
        self._check_frozen()
        if amount <= 0:
            raise NegativeAmountError(amount)
        self._balance += amount
        logger.info(f"[{self.account_id}] Deposited {amount:.2f} → balance={self._balance:.2f}")

    def withdraw(self, amount: float):
        self._check_frozen()
        if amount <= 0:
            raise NegativeAmountError(amount)
        if amount > self._balance:
            raise InsufficientFundsError(self._balance, amount)
        self._balance -= amount
        logger.info(f"[{self.account_id}] Withdrew  {amount:.2f} → balance={self._balance:.2f}")

    def freeze(self):
        self._frozen = True
        logger.warning(f"[{self.account_id}] Account frozen")


# --- Transfer context manager ---
@contextmanager
def atomic_transfer(src: BankAccount, dst: BankAccount, amount: float):
    """Transfer atomically: rolls back source withdrawal on failure."""
    logger.info(f"Transfer {amount:.2f} from {src.account_id} to {dst.account_id}")
    src.withdraw(amount)
    try:
        yield
        dst.deposit(amount)
    except BankError as e:
        logger.error(f"Transfer failed: {e} — rolling back")
        src._balance += amount  # rollback
        raise


# --- Demo ---
alice = BankAccount("ACC-001", balance=500.0)
bob   = BankAccount("ACC-002", balance=100.0)
eve   = BankAccount("ACC-003", balance=200.0)
eve.freeze()

operations = [
    ("Deposit 200 to Alice",       lambda: alice.deposit(200)),
    ("Withdraw 100 from Alice",    lambda: alice.withdraw(100)),
    ("Withdraw 1000 from Bob",     lambda: bob.withdraw(1000)),
    ("Deposit -50 to Bob",          lambda: bob.deposit(-50)),
    ("Deposit to frozen Eve",       lambda: eve.deposit(10)),
    ("Transfer 150 Alice→Bob",     lambda: atomic_transfer(alice, bob, 150).__enter__()),
]

print("="*55)
for desc, op in operations:
    print(f"\n▶ {desc}")
    try:
        op()
    except InsufficientFundsError as e:
        logger.error(f"  Insufficient funds: {e}")
    except NegativeAmountError as e:
        logger.error(f"  Negative amount: {e}")
    except FrozenAccountError as e:
        logger.error(f"  Frozen account: {e}")
    except BankError as e:
        logger.error(f"  Bank error: {e}")

print(f"\n{'='*55}")
print(f"Final balances — Alice: {alice.balance:.2f}  Bob: {bob.balance:.2f}")
